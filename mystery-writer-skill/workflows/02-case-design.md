# Workflow 02: 案件设计 (Case & Mystery Design)

## 1. 触发条件
* 入口判断为新故事构思阶段，或用户要求设计、细化具体案件。

## 2. 前置依赖
* 调用 `knowledge/genres/`（确认并对齐流派底色，如：本格解谜、社会派、硬汉派、新本格等）。

## 3. 核心处理逻辑
遵循推理小说**“从结果逆推原因”**的底层逻辑。AI 必须在后台首先建立“全知视角（Omniscient View）”，严禁跟随主角视线边想边编：
1. **死者与终局（The Corpse）**：谁死了？在何时、何地、因何致死？（确立第一现场与发现现场）。
2. **动机与嫌疑（The Motive）**：谁有核心动机？谁具备作案时间？（确立初步嫌疑人谱系）。
3. **证据链闭环（The Evidence）**：排除所有伪解答后，能够一锤定音锁定真凶的**“唯一铁证（Smoking Gun）”**是什么？
4. **⚠️ 逻辑强压测试**：检查凶手在没有主角介入的情况下，其行为链条是否符合正常人类的智商与本能。

## 4. 输出规范
AI 完成分析后，必须严格按照以下三段式结构进行输出，严禁模糊带过：

### ①【案件基石建立】
* 明确输出并填写以下两张核心卡片（使用 Markdown 表格或区块）：
  * `templates/mystery/case-card.md`（案件核心卡：死因、现场、唯一铁证）
  * `templates/mystery/suspect-map.md`（嫌疑人谱系：人设、显性动机、隐性动机、不在场证明）

### ②【底层逻辑宣告】
* 明确告知用户：**当前案件的底层逻辑（凶手-动机-铁证）已锁死。**

### ③【下一步分流引导】
* 给出当前阶段的承接引导，提示用户选择下一步路由：
  * **分支 A**：如果案件涉及特殊手法、密室、或复杂误导，调用 `03-trick-design.md` 进入诡计包装。
  * **分支 B**：如果属于纯逻辑流、心理流或社会派，跳过诡计，直接调用 `04-outline.md` 进入大纲排布。
